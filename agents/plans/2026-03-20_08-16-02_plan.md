# Implementation Plan: Parent/Guardian Contact Fields

## Goal Description
Update the student registration form and database schema to collect multiple structured parent/guardian contacts, specifically Name, Phone Number, and Email for each parent, leveraging an array of inputs with a "+" button.

## Proposed Changes

### 1. Database Schema
#### [MODIFY] src/lib/server/db/schema.ts
- Add `parentNames: text("parent_names")` to the `students` table schema definition to complement existing `parentPhone` and `parentEmails` columns.
- Ensure all three columns store comma-separated lists to maintain consistency.

### 2. Database Migration
- Build and apply a Drizzle schema migration for the new field (`npx drizzle-kit generate` and `wrangler d1 migrations apply`).

### 3. Student Registration Form
#### [MODIFY] src/routes/register/+page.svelte
- Instead of simple string variables, initialize a Svelte `$state` array of objects (e.g., `parents = [{name, phone, email}]`) based on the split values of `data.student.parentNames`, `parentPhone`, and `parentEmails`.
- Replace the existing `parentEmails` and `parentPhone` inputs with an `{#each parents as parent, i}` loop containing three fields: Name, Phone, Email.
- Assign form names `parentNames`, `parentPhones`, and `parentEmails` respectively so they group functionally via `FormData`.
- Use "+" to push a new blank object and "x" to remove one.
- Apply Bootstrap utility styling (e.g., `d-flex gap-2 flex-wrap`, etc.) to keep it responsive and clean.

### 4. Form Action Handler
#### [MODIFY] src/routes/register/+page.server.ts
- Change `formData.get(...)` to `formData.getAll("parentNames")` (and phones, emails).
- Filter out empty submissions and ensure indices line up, then join them back into comma-separated strings.
- Add `parentNames` to the `db.update` and `db.insert` queries.

### 5. Admin Student Overview
#### [MODIFY] src/routes/(secure)/admin/students/+page.server.ts
- Include `parentNames: table.students.parentNames` in the database select query so it matches the other fetched parent fields.

## Verification Plan
### Automated Tests
- Run `npm run check` to verify SvelteKit type checking.

### Manual Verification
1. Run `npm run db:generate` and local migrations.
2. Complete the form under `/register` and verify that the UI nicely formats three inputs per parent.
3. Test adding and removing parent objects natively on the client.
4. Save the profile and verify that reloading the page populates the fields properly.
5. Visually inspect the database to ensure comma-separated values save correctly.
