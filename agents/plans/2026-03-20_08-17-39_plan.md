# Plan: Add current grade and gender to registration form

## Goal
Add `currentGrade` and `gender` to the student registration form and database schema as requested by the user.

## Steps
1. **Schema Update**:
   - File: `src/lib/server/db/schema.ts`
   - Action: Add `currentGrade: text("current_grade")` and `gender: text("gender")` to the `students` table.
2. **Database Migration**:
   - Run `npm run db:generate` to generate Drizzle SQL migration.
   - Run `npm run db:migrate:local` to apply the migration to the local D1 database.
3. **Backend Logic**:
   - File: `src/routes/register/+page.server.ts`
   - Action: Extract `currentGrade` and `gender` from `formData`. Add validation (e.g. they are required). Pass them to the DB insert/update statement.
4. **Frontend Form**:
   - File: `src/routes/register/+page.svelte`
   - Action: Add a select dropdown for `currentGrade` (e.g., 9, 10, 11, 12, or Freshman, Sophomore, Junior, Senior).
   - Action: Add a select or radio button group for `gender`.

## Verification
- Load the `/register` page and verify the new fields appear.
- Submit the form and ensure data is properly saved to the database without errors.
- Ensure type safety rules are satisfied (e.g., Drizzle types).
- **Testing**: Write a unit/e2e test to verify the updated form submission logic handles 'currentGrade' and 'gender' correctly.
